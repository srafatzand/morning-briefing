# Morning Briefing — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a personal, password-protected daily news briefing web app that auto-generates AI-curated briefings nightly and serves them through a minimal editorial interface.

**Architecture:** Next.js 15 (App Router) handles both frontend and API in one repo. A Vercel Cron Job calls `/api/generate` nightly; that route uses the Anthropic SDK with built-in web search to generate and store a structured briefing in PostgreSQL (Neon). Next.js middleware enforces a PIN gate on all routes except `/login`.

**Tech Stack:** Next.js 15, TypeScript, Tailwind CSS, Drizzle ORM, PostgreSQL (Neon serverless), Anthropic SDK (`@anthropic-ai/sdk`), `jose` (JWT cookies), Vercel Cron Jobs.

| Technology | Choice | Why |
|---|---|---|
| Framework | Next.js 15 App Router | Full-stack in one repo — API routes + React UI. Server Components fetch data on the server with no extra API calls. |

| AI | Anthropic SDK + `web_search_20250305` | Built-in web search; no third-party search API needed. Claude handles search + summarization in one call. |
| Scheduler | Vercel Cron Jobs | Built-in, zero-config, triggers a Next.js API route on a cron schedule. Free on Vercel hobby. |
| Deployment | Vercel | First-class Next.js support, cron jobs, env vars, preview deploys. |

## 3. Database Schema

```sql
CREATE TABLE briefings (
  id            SERIAL PRIMARY KEY,
  date          TEXT NOT NULL UNIQUE,      -- "2026-03-31" (YYYY-MM-DD)
  generated_at  TIMESTAMP NOT NULL DEFAULT NOW(),
  model_used    TEXT NOT NULL,             -- "claude-sonnet-4-20250514"
  content       JSONB NOT NULL,            -- structured briefing (see below)
  duration_ms   INTEGER                    -- generation time in ms
);

CREATE INDEX briefings_date_idx ON briefings(date DESC);
```

**`content` JSONB shape:**
```json
{
  "sections": [
    {
      "topic": "AI & Tech",
      "stories": [
        {
          "headline": "Story headline here",
          "summary": "4-6 sentence plain-language summary.",
          "sources": ["https://reuters.com/..."]
        }
      ]
    }
  ]
}
```

**Drizzle schema (`src/lib/db/schema.ts`):**
```typescript
import { pgTable, serial, text, timestamp, jsonb, integer } from 'drizzle-orm/pg-core';

export const briefings = pgTable('briefings', {
  id: serial('id').primaryKey(),
  date: text('date').notNull().unique(),
  generatedAt: timestamp('generated_at').notNull().defaultNow(),
  modelUsed: text('model_used').notNull(),
  content: jsonb('content').notNull(),
  durationMs: integer('duration_ms'),
});

export type Briefing = typeof briefings.$inferSelect;
export type NewBriefing = typeof briefings.$inferInsert;
```

## 4. API Routes

| Route | Method | Auth | Description |
|---|---|---|---|
| `/api/auth/login` | POST | None | Body: `{ pin }`. Compares to `PIN_CODE` env. Sets `briefing_session` httpOnly cookie (JWT, 24h). Returns 200 or 401. |
| `/api/auth/logout` | POST | None | Clears `briefing_session` cookie. Returns 200. |
| `/api/briefings` | GET | Middleware | Returns `[{ id, date, generatedAt }]` sorted newest-first. |
| `/api/briefings/[date]` | GET | Middleware | Full briefing for given `date` (YYYY-MM-DD). 404 if missing. |
| `/api/generate` | POST | `CRON_SECRET` header | Called by Vercel Cron. Header: `Authorization: Bearer <CRON_SECRET>`. Generates today's briefing if not already done. 409 if exists. |

## 6. Briefing Generation Logic

### System Prompt

```
You are a personal news briefing assistant for a 19-year-old pre-med CS student
in Montreal with entrepreneurial interests at the intersection of tech, medicine,
and startups. Today's date is {DATE}.

Generate a daily briefing covering these topics:
- AI & Tech (2 stories)
- Biotech & Medicine (2 stories)
- Geopolitics (2 stories)
- Iran News (1 story)
- Canadian / Quebec News (1 story)
- MMA & Combat Sports (1 story)
- Claude / Anthropic News (1 story)

STRICT SOURCE RULES:
- For geopolitics and Iran news, use ONLY: Reuters, Al Jazeera, New York Times, Washington Post, BBC, CBC, The Economist
- For geopolitics, include both Western (Reuters, BBC, NYT, WaPo, The Economist) and non-Western (Al Jazeera) perspectives where available
- For AI & Tech and Biotech & Medicine, you have more flexibility but prioritize reputable sources, including but not limited to scientific journals like: Nature, Science, JAMA, The New England Journal of Medicine (NEJM)
- Do NOT cite Twitter, Reddit, Substack, blogs, or any unaccredited source


CONTENT RULES:
- Stories must be from the last 48 hours — genuinely newsworthy TODAY
- Each summary is 4-6 sentences, plain direct language
- Filter for relevance to someone building at the intersection of tech, medicine, startups
- For Claude/Anthropic: search specifically for Anthropic announcements or research

OUTPUT: Return ONLY a valid JSON object. No markdown fences, no preamble.
{
  "sections": [
    {
      "topic": "AI & Tech",
      "stories": [
        { "headline": "...", "summary": "...", "sources": ["url"] }
      ]
    }
  ]
}
```

### Claude API Call (`src/lib/briefing-generator.ts`)

```typescript
import Anthropic from '@anthropic-ai/sdk';
import { parseBriefingJSON } from './briefing-parser';
import { db } from './db';
import { briefings } from './db/schema';

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

export async function generateBriefing(date: string) {
  const start = Date.now();

  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 4096,
    tools: [{
      type: 'web_search_20250305' as const,
      name: 'web_search',
      max_uses: 25,
    }],
    system: buildSystemPrompt(date),
    messages: [{ role: 'user', content: "Generate today's briefing now." }],
  });

  // Response has multiple blocks (tool use rounds, then final text)
  // Find the last text block
  const textBlock = [...response.content].reverse().find(b => b.type === 'text');
  if (!textBlock || textBlock.type !== 'text') {
    throw new Error('Claude did not return a text response');
  }

  const content = parseBriefingJSON(textBlock.text);

  const [saved] = await db.insert(briefings).values({
    date,
    modelUsed: 'claude-sonnet-4-20250514',
    content: content as any,
    durationMs: Date.now() - start,
  }).returning();

  return saved;
}
```

### Output Parser (`src/lib/briefing-parser.ts`)

```typescript
import type { BriefingContent } from '@/types';

export function parseBriefingJSON(raw: string): BriefingContent {
  // Strip markdown fences if Claude wrapped the JSON
  const cleaned = raw
    .replace(/^```json\s*/i, '').replace(/^```\s*/i, '').replace(/\s*```$/, '').trim();

  let parsed: unknown;
  try {
    parsed = JSON.parse(cleaned);
  } catch (e) {
    throw new Error(`Failed to parse briefing JSON: ${e}. Raw: ${raw.slice(0, 200)}`);
  }

  if (!parsed || typeof parsed !== 'object' || !Array.isArray((parsed as any).sections)) {
    throw new Error(`Invalid briefing shape. Got: ${JSON.stringify(parsed).slice(0, 200)}`);
  }

  return parsed as BriefingContent;
}
```

**`vercel.json`** (in project root):
```json
{
  "crons": [{ "path": "/api/generate", "schedule": "0 7 * * *" }]
}
```
7:00 UTC = 3:00am EST / 2:00am EDT — covers both Eastern time zones.

```typescript
const auth = req.headers.get('authorization');

  return Response.json({ error: 'Unauthorized' }, { status: 401 });
}
```

**Manual trigger during development:**
```bash
npm run generate          # runs scripts/generate-now.ts via tsx
```

**How it all connects:**
1. Every day at 7am UTC, Vercel's infrastructure sends `POST /api/generate` with the cron secret in the Authorization header
2. The route verifies the secret, checks if today's briefing already exists (idempotent), then calls `generateBriefing(today)`
3. The briefing is stored in the DB; next time someone visits `/today`, it's already there

## 8. Auth Flow

```
User visits /today
      ↓
middleware.ts checks for 'briefing_session' cookie
      ↓
Cookie missing or invalid JWT → redirect to /login?from=/today
      ↓
User enters 4-digit PIN in PinForm
      ↓
POST /api/auth/login { pin: "1234" }
      ↓
Server compares pin === process.env.PIN_CODE
      ↓
Match:    sign JWT with jose → set httpOnly cookie → return 200
No match: return 401 → PinForm shows "Incorrect PIN"
      ↓
Client redirects to ?from path (default: /today)
```

**Cookie spec:**
- Name: `briefing_session`
- HttpOnly: `true` (JS cannot read it — prevents XSS theft)
- Secure: `true` in production
- SameSite: `lax`
- MaxAge: `86400` (24 hours)
- Value: JWT signed with `JWT_SECRET` env var

**Middleware matcher** (protects everything except login + auth API):
```typescript
export const config = {
  matcher: ['/((?!login|api/auth|_next/static|_next/image|favicon.ico).*)'],
};
```

  ```typescript
  import { SignJWT, jwtVerify } from 'jose';

  const secret = new TextEncoder().encode(process.env.JWT_SECRET!);

  export async function createSessionToken() {
    return new SignJWT({ authenticated: true })
      .setProtectedHeader({ alg: 'HS256' })
      .setIssuedAt()
      .setExpirationTime('24h')
      .sign(secret);
  }

  export async function verifySessionToken(token: string) {
    try {
      await jwtVerify(token, secret);
      return true;
    } catch {
      return false;
    }
  }

  export function isPinValid(pin: string) {
    return pin === process.env.PIN_CODE;
  }
  ```

  export async function POST(req: NextRequest) {
    const { pin } = await req.json();
    if (!isPinValid(pin)) {
      return NextResponse.json({ error: 'Invalid PIN' }, { status: 401 });

  import { NextRequest, NextResponse } from 'next/server';
  import { verifySessionToken } from '@/lib/auth';

  export async function middleware(req: NextRequest) {
    const token = req.cookies.get('briefing_session')?.value;
    const isValid = token ? await verifySessionToken(token) : false;
    if (!isValid) {
      const url = new URL('/login', req.url);
      url.searchParams.set('from', req.nextUrl.pathname);
      return NextResponse.redirect(url);
    }
    return NextResponse.next();

#### Task 3.3: Write generate API route
- [ ] Create `src/app/api/generate/route.ts`:
  ```typescript
  import { NextRequest, NextResponse } from 'next/server';
  import { generateBriefing } from '@/lib/briefing-generator';
  import { db } from '@/lib/db';
  import { briefings } from '@/lib/db/schema';
  import { eq } from 'drizzle-orm';

  export async function POST(req: NextRequest) {
    const auth = req.headers.get('authorization');
    if (auth !== `Bearer ${process.env.CRON_SECRET}`) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
    }

    const today = new Date().toISOString().split('T')[0];

    const existing = await db.query.briefings.findFirst({
      where: eq(briefings.date, today),
    });

    if (existing) {
      return NextResponse.json({ message: 'Already generated', date: today }, { status: 409 });
    }

    const briefing = await generateBriefing(today);
    return NextResponse.json({ success: true, date: briefing.date });
